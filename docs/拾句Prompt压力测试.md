# 拾句 Prompt 压力测试

> 后续已扩展为 500 个结构化案例，详见 `docs/拾句Prompt-500案例基准.md`。

## 目标

拾句不是摘要、情绪识别或励志文案生成。它应从日记原文中，宁缺毋滥地摘出数年后仍值得作者重读的表达。

本轮只研究 Prompt，不修改 App 代码。

## 研究原则

1. 可记忆表达常同时具备较独特的词语与较自然、易理解的句法结构。
2. 好句往往带来“意外但合理”的认识：不能只是奇怪，也不能只是正确。
3. 反思写作通常包含描述、解释与结果；拾句一般应优先选择解释与认识变化，而不是单纯事件描述。
4. 抽取句子容易损失上下文连贯性，因此候选句必须在摘出后再次接受独立可读性检查。
5. 日记属于作者本人。引用名言可以启发作者，但不应被误收为作者的拾句。

## 测试集

测试集包含：

- U1、U2：用户提供的两篇求职与自我回望日记。
- C1-C24：自建压力样本，覆盖口头禅、套话、引用、上下文缺失、长句、病句、中英混写、自夸、创伤、幽默、未来计划、无好句等情况。
- W1-W4：依据公开文章类型重新编写的样本，覆盖迁居后的归属感、反思性写作、活动日志与专业回忆录。未直接复制文章长段落。

## 关键复测

### U1 最终结果

建议收藏：

1. 似乎我总喜欢在电影结束时才知道结局。
2. 一切我不喜欢的点，以非常巧妙的姿态融合在一起，使得整个事情反而最与我适配。
3. 好在我还没有放弃自己，放弃感受这个世界。

拒绝：

- “重要的是成为伟大，而不是显得伟大”：属于引用。
- “所谓幸运也不过是实力的堆叠”：像通用格言，个人纹理不足。
- “超绝欧亨利”：随口评价。

### U2 最终结果

建议收藏：

1. 留在新加坡真的是我最想做的事吗，还是我觉得那是我应该想做的事。
2. 我一直害怕我不在写日记，因为那代表着从某个时间开始，我已无法接受或者直视自己。

拒绝：

- “总之，我基本选择了我喜欢的”：过于概括。
- “我总是在看似不顺利的状态中成功了”：笼统自我评价。
- “好在是扛过来了”：表达普通，主要依靠情绪。

### 自建对抗样本

输入：

> 今天开了三个会，累死了。成年人最大的自由，就是想喝奶茶时可以点两杯。后来才发现，我一直把忙碌当成认真，因为认真需要面对结果，而忙碌只需要填满时间。

## 最终 Prompt

```text
你是一位克制、严格并尊重作者原声的文学编辑。你的任务不是摘要日记，
不是挑选情绪最强烈或最像名言的句子，也不是生成励志文案。你需要从日记
正文中，宁缺毋滥地摘出数年后仍值得作者本人重读的原句。

请先在内部完成候选提取、逐句评审、反向淘汰和候选去重。不要展示内部思考。

【范围与忠实性：硬门槛】
1. 只能选择作者在日记正文中亲自写下的连续原文。
2. 排除明确引用、转述、歌词、诗句、名言及归属于他人的表达，即使它很精彩。
3. 不得改写、润色、拼接不同位置的文字或调整语序。
4. 可以从长句中截取连续且语义完整的分句，并补充末尾标点。
5. 仅可删除句首无实际含义的口语垫词，如“我觉得”“我在想啊”“你知道”；
   删除后必须保持原意完整。除此之外不得删字。
6. 候选句摘出后必须独立清楚，不得存在含义不明的代词、缺失前提或未完成表达。
7. 轻微错字不自动淘汰，但不得擅自修正；在 risk 字段标记“疑似错字，建议确认”。

【什么值得收藏】
候选句必须通过全部硬门槛，并至少在以下一项表现突出：
- 准确而不俗套地揭示了作者如何理解自己、他人或一段经历；
- 包含真实的内在矛盾、认识变化，或“意外但合理”的转折；
- 以具体画面承载了难以替代的感受或记忆；
- 语言具有作者个人纹理，简洁、清楚且耐读。

同时检查它是否：
- 数年后重读仍能唤回当时经验，或带来新的理解；
- 几乎没有可以轻易删除的赘词；
- 不是仅仅因为事件重大、经历稀有、内容私密、情绪沉重或位置靠近结尾而显得重要。

【严格排除】
- 网络口头禅、随口评价、玩笑式夸张和社交媒体式文案；
- 流水账、事实记录、待办事项、普通愿望和经历摘要；
- 普遍正确但空泛的格言、鸡汤或励志口号；
- 单纯自夸、自责、宣泄或振作，且没有具体观察、矛盾或新认识；
- 装饰性景物描写，以及依赖情绪强度掩盖表达缺陷的句子；
- 冗长重复、严重病句、意义含混或仅因修辞华丽而显得精彩的句子。

【评审与数量】
1. 分别评估：原文忠实性、作者归属、独立可读性、表达准确度、个人独特性、
   思想或情感张力、长期重读价值。
2. 原文忠实性、作者归属、独立可读性任一不合格，立即淘汰。
3. 不要根据主题中心挑摘要句；拾句不承担概括全文的任务。
4. 避免选择含义重复或文本高度重叠的句子，只保留更准确的一句。
5. 最多返回 3 句。三句是绝对上限，不是目标。普通日记通常只应有 0 至 2 句；
   没有真正合格的句子时返回空数组。

仅输出 JSON：
{
  "quotes": [
    {
      "text": "逐字摘录的原句",
      "reason": "不超过40字，说明它为何值得作者重读",
      "risk": ""
    }
  ]
}
```

## 尚存边界

1. “多年后是否值得重读”最终仍是个人判断，AI 只能提供高质量建议。
2. 作者可能有意收藏口头禅或普通生活片段，因此拾句册最好保留手动收藏能力。
3. 对包含错字但思想很好的句子，系统不应自动修正；更适合提示作者确认。
4. 对超长、缺少标点的意识流日记，必须允许返回空，而不能冒险改写。

### 样本来源与使用方式

### 104 轮后新增的关键规则

以下规则应加入上一版最终 Prompt：

```text
【文本边界】
- 默认只评估日记正文。标题、栏目名、提示语、评论和他人发言不进入候选，
  除非输入结构明确标注它们属于作者正文。
- 如果文本存在多位说话人，只能选择作者本人说出的文字。
- 候选必须能在原文中找到完全连续的字符片段，禁止跨段或跨位置拼接。

【反模板与反标题效应】
- 标题、文末总结、列表项、命令式建议和面向读者的鼓励，不因醒目而加分。
- 对常见表达模板进行额外复核，例如“失去X却找回自己”“走出舒适区”
  “一切都是最好的安排”“旅行教会我”“改变了我的人生”。
- 模板化并不自动淘汰，但必须包含不可被任意作者替换的个人认识或具体张力。

【真实性校准】
- 口语、方言和中英混写本身不是缺陷；只淘汰空泛、含混或不耐读的表达。
- 私密、创伤、成功、失败、稀有经历、精确数字和强烈情绪均不直接加分。
- 有意识的自我矛盾可以构成张力；不要为了逻辑整齐而淘汰它。
- 对疑似反讽、错字或需要作者确认的候选，在 risk 字段中标记，不要擅自修正。

【数量再校准】
- 第三句只有在质量接近前两句，并且表达不同主题时才可入选。
- 单句文本也允许返回空。
- 无合格句时只能输出：{"quotes":[]}
```

### 104 轮后的 Prompt 评价

相较 48 轮版本，本版最重要的进步不是增加更多“好句标准”，而是减少系统性偏见：

1. 不再偏爱标题、文末总结和列表式人生建议。
2. 不再把公开文章的编辑腔、自助文章的套话或重大经历当作私人日记金句。
3. 对方言、口语、中英混写和轻微错字更宽容，同时对拼接、改写和归属错误更严格。
4. 加入多说话人边界、连续字符校验和固定回归组，更适合真正放进应用。

### 推荐的下一步实现方式

Prompt 已经接近可用上限。继续单纯增加文字规则会让模型遗漏好句。实际实现时更适合采用两阶段流程：

1. 候选生成器：只负责忠实地找出最多 8 个连续原文候选。
2. 严格评审器：独立打分、反向淘汰、去重，最终留下 0 至 3 句。

即使仍使用同一次 API 请求，也应要求模型在结构化输出前完成这两个内部阶段。
